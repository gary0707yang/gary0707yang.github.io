一、引言

很多网站都有引导用户发生转化的相对固定的路径，比如电商类网站的首页 - 产品页 - 加入购物车 - 结算 - 下单完成，这样的路径我们称之为转化路径。经验表明，进入了转化路径的用户，在各个转化节点上会存在不同程度的流失，如果用图表来说明从前一步到达后一步的数据，大致会呈现一个漏斗形状。如果我们能知道用户在转化路径上各个节点的流失情况，那么我们便可以针对性地对转化路径进行优化，从而提高转化率。

二、转化漏斗的设置

在GA 中，转化漏斗功能位于转化 - 目标 -渠道可视化 这里，GA 英文版中这一功能被称为Funnel Visualization，即漏斗的可视化。要想在这里看到转化漏斗的数据，必须先在目标那里设置好。目标的设置位于管理 - 查看 - 目标，要想使用转化漏斗功能，必须要启用目标设置中的渠道，一个设置demo如下图所示：

这里需要注意的有三点：

1）只有目标网址类型的目标才支持渠道的设置，即才支持转化漏斗功能；

2）渠道中各个步骤的url 匹配规则总是和目标网址那里选定的规则一致，比如，如果目标网址那里你选用的是正则表达式的规则匹配，那么步骤的url 记得也要使用正则表达式来写；

3）步骤1 后边的能否跳过这个选项会决定最终参与转化漏斗计算的数据，默认是否，即可跳过，如果你选择了是，那么只有当一次访问当中真的经过了步骤1，才能算作真正的进入了转化漏斗，才能参与转化漏斗数据的计算。

三、转化漏斗的报告

在GA 中转化漏斗报告位于渠道可视化，为了方便说明，以下一律使用渠道可视化来表述。渠道可视化报表主要分为两部分，上半部分是目标的转化率和目标转化率的折线图，下半部分即是转化漏斗图。我们这里主要关注转化漏斗图，一个典型的转化漏斗图如下所示：

转化漏斗又可分为三个部分，左边是渠道进入点，表示的是用户在进入转化渠道前浏览的那个页面，如果着陆页即为转化渠道中的页面，那么渠道进入点显示为(entrance)，渠道进入点最多展示top 5页面的数据，绿色小三角即表示进入渠道；中间为转化漏斗，清楚地表示了用户在各个转化节点的流失数据，转化节点间流失的百分比数据不仅体现在条形中绿色和红色的占比，也体现在转化节点间漏斗的大小；最右边部分即为渠道退出点，展示了用户从转化渠道退出后去向了哪些页面，如果渠道中某步骤的有效的浏览即为退出页面，那么渠道退出点将以(exit)表示。

四、转化漏斗数据计算的规则

渠道可视化中数据的计算比较复杂，要理解这里边数据的计算逻辑，首先需要理解渠道进入点和渠道退出点这两个维度。

为了方便说明，我们假设用户设定的目标页面为Goal，各转化节点为Step1 -> Step2 -> Step3 -> Step4 ，我们看到，这四个步骤的Step Number 依次递增。

  1. 渠道进入点和渠道退出点

渠道进入点表示的是一次session 中首次匹配到转化渠道页面时的前一个页面，之前已经说到，如果渠道页面即为着陆页，那么渠道进入点为(entrance)。这里需要特别注意的是，渠道进入点总是被分配给当前session 中Step Number 最小的那个。比如，一次用户的浏览为：entrance -> Other1 -> Step2 -> Step1 -> Step3 -> exit，在渠道可视化报告中，渠道进入点为Other1，将展示在Step1 的左边，而不是在Step 2的左边。

渠道退出点表示的是一次session 中最后一个非重复且能匹配转化渠道页面的后一个页面，如果该页面即为退出页面，那么渠道退出点为(exit)。比如，一次用户的浏览为：entrance -> Other1 -> Step1 -> Step2 -> Step1 ->exit，在渠道可视化报告中，这次的渠道退出点为Step1，渠道进入点为Other1，并且Step1 将最终出现在Step2的右侧（如果在top5渠道退出点的话）。

  1. 一次session 中每个Step都只会计算一次

不管一次session 中某个Step 被经过多少次，最终每个Step 最多计数一次，这个有点类似于唯一身份浏览量的概念。

  1. 被跳过的Step 会被自动填充

比如，一次session 中由Step1 -> Step3，那么最终出现在渠道可视化报表中的是Step1 -> Step2 -> Step3，被跳过的Step2 被自动填充上来了。

  1. 渠道转化率和转化率

某目标的渠道转化率 = 该目标的达成次数/进入该目标渠道的次数 * 100%

某目标的转化率 = 该目标的达成次数/总的访问次数 * 100%

  1. 一个相等关系

渠道进入次数 =目标达成次数 + 渠道退出次数

即转化漏斗左边所有绿色三角旁边数字之和要等于目标达成的次数加上右边红色三角旁边数字之和

五、总结

渠道可视化功能关注的是各转化节点间的流失情况，它并不会告诉你用户真实的行为轨迹，如果需要知道用户在转化节点上的回环情况，可以查看下目标下的目标流报告；如果需要查看用户达成目标前经过的其他非转化节点页面，可以看下逆向目标路径报告。

但是，渠道可视化报告能够从宏观上让你把握下用户在转化路径上的流失，通过分析渠道进入点和渠道退出点还能对转化渠道设置进行诊断，从而提高各渠道节点的通过率，并最终提高渠道转化率以及ROI。
